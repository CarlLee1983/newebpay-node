# 安全性政策 (Security Policy)

## 支援版本 (Supported Versions)

目前我們提供以下版本的安全性更新支援：

| 版本 | 支援狀態 |
| :--- | :--- |
| 1.x.x | ✅ 支援中 |
| < 1.0.0 | ❌ 不再支援 |

## 漏洞回報 (Reporting a Vulnerability)

如果您發現本專案有任何安全性漏洞，請**不要**使用公開的 issue tracker 回報。

請將漏洞詳情發送至我們的安全團隊信箱：
**carllee0520@gmail.com**

### 回報內容建議

為了協助我們快速評估與修復漏洞，請在郵件中提供以下資訊：

1. **漏洞類型**：例如 SQL Injection, XSS, 加密實作瑕疵等。
2. **影響範圍**：受影響的 SDK 版本、模組或功能。
3. **重現步驟**：請提供最小可重現範例 (Minimal Reproducible Example) 或攻擊概念驗證 (PoC)。
4. **建議修復方式**（如果有）。

### 處理流程

1. 我們會在收到回報後的 48 小時內回覆確認。
2. 安全團隊將評估漏洞嚴重性並著手進行修復。
3. 在修正釋出之前，請您協助保密漏洞細節。
4. 修復完成後，我們會發布新版本並在 Release Notes 中公開致謝（如果您同意）。

感謝您協助維護本專案的安全性！
